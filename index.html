d34raphael

What is this?

This is a fork of the d3 (http://d3js.org/) Javascript data library to allow d3 to work as elegantly with the Raphael (http://raphaeljs.com/) vector drawing library as d3 natively does with SVG.

How can I use this?

Look at the source of the example projects above.

  1. Patched Raphael (see the diff)
  2. d3 with Raphael support
  3. (For IE8) Sizzle selector library
  4. (For IE8) IE Array.prototype and CSS fixes
  5. Write Raphael compatible d3

Why?

Because Internet Explorer <= 8 is still important to many people. IE8 and below doesn't natively support SVG, but does support an older Microsoft vector drawing standard called VML. Raphael provides an abstraction layer between your Javascript code and the vector art implementation of the browser.

Specifically, Raphael does VML for IE8 and SVG for everyone else.
